- Decision: ITERATE
- Justification: The current pipeline is conceptually strong but suffers from a "linear bottleneck" (PCA) that likely discards the non-Gaussian information essential for detecting subtle hydro-code mismatches. The proposed changes—specifically removing PCA to allow the MAF to learn directly from the high-dimensional WST feature space and implementing a more rigorous "nuisance-invariance" test—directly address the "Structural Robustness" and "Reviewer Confidence" criteria. Since the current infrastructure (WST + CNF) is already functional and efficient, the ROI of these refinements is high: they transform the model from a "power-spectrum-like" detector into a true non-Gaussian anomaly detector. Moving to a new project would abandon a high-potential architecture just as it reaches the critical stage of fine-tuning for robustness.
