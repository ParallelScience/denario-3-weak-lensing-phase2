The VCSF pipeline is conceptually sound, but the current results reveal a critical methodological vulnerability: the reliance on PCA for dimensionality reduction prior to density estimation. While PCA captures 97.97% of the variance, it is a linear transformation that assumes Gaussianity in the feature space. The non-Gaussian structural information (peaks, voids, filaments) that you correctly identified as the target signal is likely encoded in the higher-order moments of the WST coefficients, which PCA explicitly discards as "low variance" components.

**Critical Weaknesses & Gaps:**
1. **Information Loss via PCA:** By reducing 150 WST features to 75 PCs, you are likely discarding the very non-Gaussian "tails" of the distribution that distinguish different hydro-code implementations. If the OoD signal is subtle and non-Gaussian, it is almost certainly hidden in the components PCA treats as noise.
2. **Synthetic Validation Bias:** Your synthetic validation (Fourier notch filter) is a "straw man" perturbation. It tests if the model can detect a global power-spectrum suppression, which is a much easier task than detecting the subtle, localized, and non-linear baryonic feedback differences between two hydro codes. This explains the "exquisite sensitivity" you observed, which may not translate to the actual test set.
3. **Regressor Over-confidence:** You report an RMSE for $T_{AGN}$ of 0.33. Given the range [7.2, 8.5], this is a significant error. If the regressor is wrong, the CNF is conditioned on the wrong $\theta$, leading to a high NLL that is a false positive (nuisance-driven) rather than a true OoD signal.

**Actionable Recommendations:**
1. **Eliminate PCA:** The MAF (Masked Autoregressive Flow) is specifically designed to handle high-dimensional, correlated data. Feed the raw 150 WST features directly into the MAF. If compute budget is a concern, use a smaller number of layers in the MAF rather than reducing the feature dimensionality.
2. **Refine Synthetic Validation:** Replace the Fourier notch filter with a "Hydro-Swap" proxy. Use a subset of your training data generated with a different (if available) or perturbed subgrid model (e.g., vary $T_{AGN}$ to its extreme bounds) to see if the model flags these as OoD. If it does, your model is not yet invariant to nuisance parameters.
3. **Improve Nuisance Marginalization:** Instead of just sampling $\theta$ from the regressor, incorporate the regressor's uncertainty directly into the CNF training. Use a "Variational Information Bottleneck" approach where the CNF is trained to maximize the evidence lower bound (ELBO) on the features, effectively forcing the model to learn a representation that is invariant to the nuisance parameters $\theta$.
4. **Feature Importance:** Instead of `mutual_info_regression` (which is linear-centric), use a Random Forest regressor to rank feature importance. This will capture non-linear dependencies between WST coefficients and the physical parameters, providing a more robust feature selection for the non-Gaussian signal.

**Future Iteration Focus:**
Stop optimizing for the synthetic validation score. Focus on the "nuisance-invariance" test: if you pass a training map with an extreme $T_{AGN}$ value, the NLL score should remain low. If it spikes, your model is failing the core requirement of the challenge.
