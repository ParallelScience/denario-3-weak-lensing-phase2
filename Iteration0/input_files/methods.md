1. **Data Preprocessing and WST Feature Extraction**: Reconstruct 2D maps (1424×176) from flattened arrays using the boolean mask (`mask[mask] = flat_pixels`). Add Gaussian noise (σ≈0.02582, applied only within the survey mask) to match the test set noise level. Compute Wavelet Scattering Transform (WST) features using `kymatio` with **J=2, L=6** on GPU (device='cuda') in **batches of 64** maps. The raw WST output per map has shape (n_coeffs, H', W'); apply **spatial average pooling** to reduce to a compact feature vector of ~200 dimensions. This gives a manageable representation while retaining the non-Gaussian baryonic structure information. *Benchmarked: GPU batch mode processes all 25,856 training maps in <1 minute. J=3/L=8 produces 849,772 features/map — too large; J=2/L=6 with pooling targets ~200 features.* Save pooled features to disk as `wst_features_train.npy` (shape: 25856 × n_features, float32).

2. **Training Set Preparation**: Z-score normalize WST features using mean and std computed over the training set. Hold out 20 of the 101 cosmological models (indices drawn randomly but fixed by seed) as the validation set — this ensures the model is evaluated on unseen cosmologies. Flatten labels to shape (25856, 5) matching the flattened map order.

3. **Conditional Normalizing Flow (CNF) Training**: Construct a Masked Autoregressive Flow (MAF) using `zuko` — specifically `zuko.flows.MAF` with 5 transforms, hidden features [256, 256]. Train to learn the conditional density $p(\text{WST features} | \Omega_m, S_8, T_{\rm AGN}, f_0, \Delta z)$ using ground-truth labels from `label.npy`. Train on GPU with Adam optimizer (lr=1e-3), batch size 512, for up to 50 epochs with early stopping on validation NLL. *Budget: ≤15 minutes.* The conditioning on all 5 parameters — including nuisances — ensures the flow captures legitimate InD variation and only flags true structural anomalies as OoD.

4. **MLP Parameter Regressor Training**: Train a 3-layer MLP (n_features → 512 → 256 → 5) to predict the five parameters from WST features. Use MSE loss, Adam optimizer, GPU. This is needed at test time because ground-truth labels are unavailable for the 10,000 test maps. Standardize both inputs (WST features) and outputs (labels) before training.

5. **Inference and OoD Scoring**: For each of the 10,000 test maps: (a) reconstruct 2D map, (b) compute WST features on GPU (batched), (c) apply spatial average pooling and z-score normalization, (d) pass through MLP to predict $\hat{\theta}$, (e) evaluate NLL = $-\log p(\text{features} | \hat{\theta})$ under the CNF. The NLL is the OoD score — higher NLL means the test map is less likely under the learned InD distribution at the predicted cosmology. Process all 10,000 maps in batches; do NOT use any inter-sample statistics at test time.

6. **Validation and Performance Estimation**: Evaluate on the 20 held-out cosmologies (their 256 realizations each = ~5,120 maps) by comparing NLL scores. Simulate OoD detection performance using cross-cosmology variance as a proxy: compute `score_phase2` using held-out vs. training distributions. Report `val_score` (partial AUC, FPR in [0.001, 0.05]).
